二、交易规则

  1. 合约名称及交替

    SZ合约，用“SZ+交割日”的形式表示，如SZ1226代表于12月26日交割的12月份SZ合约。

    老合约退市后新合约于第二交易日上午9:15分挂牌上市。

  2. 交易标的、交割

    SZ合约的交易标的为上证指数。

    交割是指在合约到期而用户又未提前选择平仓了结的情况下，平台将对到期的合约进行强制交割。

    交割时间：合约月份的最后一个周五下午15:00-15:15，以上证指数当天的收盘价为准进行交割。遇法定节假日或因单边市不能交割情况顺延。

    交割价格：以合约交割当日的上证指数收盘价为准。

  3. 合约价值、保证金收取、持仓盈亏计算

    如上表所述，平台规定一个指数点价值0.0001比特币。

    假定当前上证指数是3000点，那么一张合约的价值是：
    3000*0.0001=0.3BTC

    假定某用户使用5%的保证金（即20倍杠杆），则需要的保证金是0.015BTC最小变动价位是0.2指数点，即盈亏变动是:
    0.2*0.0001BTC=0.00002BTC

    盈亏比例变动是：0.00002/0.015=0.1333%

    波动一个指数点的盈亏是：0.0001/0.015=0.6666%

    波动150点用户本金翻翻。

    （注：以上为方便说明，未考虑交易手续费因素的影响）

  4. 报价

    报价是指您可根据自己对市场走向的判断进行开多仓（看涨）、开空仓（看跌）的操作；相应地，您也可以自主进行平多仓、平空仓的操作。但为了遏制突发异常价格波动对市价产生的不良影响，我们对您的开仓和平仓报价作出了如下强制性规定：

    （1）报价限制——防止价格短时间内大幅波动
    做多或平空时：最高可写入价为60秒前最新成交价+1%指数点
    做空或平多时：最低可写入价为60秒前最新成交价-1%指数点

    （2）固定涨跌停板制度——防止合约价格短时间内过度偏离指数价格
    SZ合约每日的涨跌停板幅度为5%，涨跌基准为前一个交易日平台内的成交均价（上市首日取前一个交易日的上证指数收盘价）。

  5. 止盈止损

    止盈：该功能可以使用户在价格达到一个阀值的时候，选择以某个价格“获利了结平仓”，以实现“固定盈利”的一种智能化功能。用户可以根据自己的意愿设定止盈价格，系统固定0.1%的价格差作为触发价格。

    “止盈功能”举例说明： 如果某用户以3000的指数点位开多仓，那么该用户可以设定在平台内成交点位达到3150点进行平仓操作，系统固定0.1%的差值后计算出的触发价格为3146.8（3150-3.15=3146.85，取3146.8），即当站内指数点达到3150以后，系统自动挂出3146.8的平仓单，此种情况下的平仓成为“止盈”。

    止损：该功能可以使用户在价格达到一个阀值的时候，选择以某个价格“认亏平仓”，以实现“控制亏损”的一种智能化功能。

    “止损功能”举例说明：如果某用户以3000的指数点位开多仓，则该用户可以在其成本点位3000点之下（比如2900点）设定 “挂价2900点平仓”，系统固定0.1%的差值后计算出的触发价格为2903（2900+2.9=2902.9，取2903），即当站内成交点位达到2900以后，系统自动挂出2903点的平仓单，此种情况下的平仓成为“止损”。

    需要说明的是：

    1. 为了防止您的挂单价和市价偏离过大导致系统性风险，我们限定您不必填写触发价格，我们自动取您的意向挂单价格0.1%的偏离价格作为触发价格；
    2. “止损”和“止盈”功能其中一个触发时，另一个功能失效。例如，您挂的单子“止盈”触发以后，您的单子会挂出去，此时无论该笔单子全部成交还是部分成交，系统均不再自动执行撤单操作，相应的“止损”功能失效（除非您手动撤单，再次重新设定止盈止损）；“止损”先触发时同理；
    3. 止盈和止损功能只对当前可用持仓有效；
    4. 为了降低系统风险，系统只负责挂价，不代表价格挂出后一定能成交，特别是在跳水行情发生时，所以用户应当对该功能有充分的认识，不能过度依赖该功能。
  6. SZ合约的爆仓

    SZ合约交易采用“余额爆仓制”（与“100%保证金爆仓模式”的区别），要求用户除了保证金占用余额之外，用户的合约账号可用余额必须大于浮亏的金额，一旦用户的合约账户可用余额小于浮亏金额，平台的风控人员会立即电话联系用户自行进行追加保证金余额或者通过减仓释放余额的形式进行处理，以使账户满足可用余额大于浮亏的要求。如果因通讯不畅或无人接听等不可抗力原因造成平台未能及时联系到用户或者用户未在规定的时间内进行处理，平台风控人员有权根据市价和挂单情况对用户的单子进行部分减仓（风险相对较小时）或者全部平仓（行情波动剧烈时）的办法进行处理，以免账户风险继续扩大。

    1. 爆仓判断标准：持仓浮亏＞合约可用余额

      注：合约可用余额，不含持仓保证金、不含挂单保证金，指自由金额。

    2. 风控人员联系您的唯一渠道：电话联系

      您可以自行设置是否选择电话联系，如果选择“是”，在您账户发生爆仓风险时，我们将进行电话联系；如果选择“否”，在您的账户发生爆仓风险时，我们将直接进行强平处理，不再电话联系。

    3. 您须特别注意的事项：
      1. 您须于收到我们电话通知的15分钟内进行处理，否则我们将进行强平处理，不再另行通知（如15分钟后因市价波动爆仓风险自行解除的情况除外）；
      2. 在交易日上午11:10—11:30之间发生爆仓的用户，您在收到风控人员的电话通知后，须于下午13:00开盘之前自行处理，否则下午开盘后我们即进行强平处理（上午11:30收盘时因市价波动爆仓风险自行解除的情况除外）；
      3. 在正常交易日14:45后（最后交易日14:30后）发生的爆仓，因离停盘时间较近，且存在隔夜或交割风险，我们将对爆仓账户直接进行强平处理，不再电话通知；
      4. 无论是否已经电话通知到您，因市价波动较大而使您的合约可用余额全部亏完且保证金亏损超过50%的持仓，我们直接进行强平处理，不再另行通知。
    4. 关于电话通知：

      出现下列情况中的任意一种，造成我们无法联系到您，视作您放弃对现有持仓的自行处理，我们将按照强平规则对您的持仓进行处理：

      1. 电信系统线路不畅
      2. 您的电话因所在区域无信号、欠费停机等原因导致无法接通
      3. 您的电话关机
      4. 电话打通后被您挂断
      5. 电话打通响铃结束后无人接听（只打一次）
    5. 关于强制平仓及处理规则

      强制平仓是指平台风控人员按照BZ合约风险控制规则的约定，对用户的未了结持仓进行部分或全部平仓的操作。

      强制平仓的处理办法：

      1. 保证金亏损在20%以内，我们将按照最小仓位标准对您的持仓按照市价进行强制减仓，以使您的账户脱离爆仓状态；
      2. 保证金亏损在20%-50%之间，我们我们将按照市价平掉您的一半仓位；
      3. 保证金亏损超过50%，我们将按照市价对您的持仓进行全部平仓处理；
      4. 因“单边市”出现盘中不能平掉的单子盘后通过“强制减仓”处理。
  7. 单边市风险及处理办法

    本平台所指“单边市”是指某一合约收市前30分钟内出现只有停板价格的买入（卖出）申报、没有停板价格的卖出（买入）申报，或者一有卖出（买入）申报就成交、但未打开停板价格的情形。

    SZ合约交易日出现单边市（单边市的交易日称为D交易日）：
    D交易日为最后交易日的，如果是上涨单边市，涨停价低于指数交割价，则延期进行交割，直到“站内涨停价≥指数交割价”为止再行交割；
    D交易日为最后交易日的，如果是下跌单边市，跌停价高于指数交割价，则延期进行交割，直到“站内跌停价≤指数交割价”为止再行交割；
    D日为正常交易日的，则在收盘后，实施强制减仓处理办法。
    强制减仓是指在单边市交易发生当日收盘后，平台将未能在盘中通过交易实现平仓的“爆仓”用户的持仓,以当日涨跌停板价格与该合约净持仓盈利用户按照盈亏比例从高到底的顺序依次进行撮合成交。

    强制减仓的方法：

    1. 强制减仓的价格

      强制减仓的价格为该合约D交易日的涨跌停板价格。

    2. 用户须平仓爆仓单数量的确定

      单向持仓用户，对其已爆仓的持仓按照平仓数量（最小的整数数量）进行强制减仓操作后，账户脱离爆仓状态。

      双向持仓的用户，其爆仓单先与自己的反向持仓单进行撮合，如果仍未使账户脱离爆仓状态，则其净持仓部分的平仓报单参与强制减仓撮合。

    3. 参与强制减仓撮合的盈利用户平仓范围及数量的确定

      对于净持仓盈利用户（有双向持仓的扣除反向持仓计算净持仓），按照持仓盈亏比例从高到低进行排序。将需要强制减仓的爆仓单的总数量和净盈利持仓按照盈利比例从高到低的数量进行匹配，直到爆仓账户的风险全部解除为止。

    4. 强制减仓的执行

      强制减仓于D日收市后执行，强制减仓结果作为D日用户的交易结果。
