持仓盈亏计算

BTC-Q比特币虚拟期货以比特币作为保证金，模拟与美元价格挂钩，并依据设定的杠杆系数计算盈亏，盈亏计算公式为：

盈亏金额=价格差÷杠杆系数×持仓量

其中：

1、价格差：是指市价（或交割价）与持仓均价的差额

2、杠杆系数：是指系统选定的盈亏结算基准价，每一份合约在上市之前都会公布出系统在该合约上选定的杠杆系数。

杠杆系数的选定：“新合约在上市之前的价格指数”或“距离新合约最近的那份合约在新合约上市前15:00的最新成交价”所选取的值“孰高原则”。杠杆系数的数值取100美元的整数倍值，“只入不舍”。

举例说明：

情况一：交易所正式上线交易首日

  • 假定比特币价格指数的值为400美元，则杠杆系数取500；
  • 假定比特币价格指数的值为412美元，则杠杆系数取500；
  • 假定比特币价格指数的值为499美元，则杠杆系数取500；
  • 即如果400≦数值﹤500，则杠杆系数取500；其他价格区间同理。

情况二：后续新合约的上市

假定当前最近的交易合约为1409，在1410于2013年9月30日15:15分上市之前的15:00整，1409的最新成交价为420美元，依据420美元则杠杆系数应当取值500；而此时比特币价格指数为399美元，依据399美元则杠杆系数应当取400，此时依据“孰高原则”应当取值500。

盈亏计算说明：

假定当前的杠杆系数为500，用户A以,550美元的价格开多仓（即买涨）10个比特币

假定开仓后价格涨到了555美元，那么盈利金额为： （555-550）/500×10=5/500×10=1%×10=0.1比特币

  • 如果A用1倍的杠杆，则本金为10个比特币，获利0.1个比特币，获利1%；
  • 如果A用5倍的杠杆，则本金为2个比特币，获利0.1个比特币，获利5%；
  • 如果A用10倍的杠杆，则本金为1个比特币，获利0.1个比特币，获利10%；
  • 如果A用20倍的杠杆，则本金为0.5个比特币，获利0.1个比特币，获利20%；
  • 如果A用50倍的杠杆，则本金为0.2个比特币，获利0.1个比特币，获利50%；

假定开仓后价格跌到了545美元，那么亏损金额为： （550-545）/500×10=5/500×10=-1%×10=-0.1比特币

  • 如果A用1倍的杠杆，则本金为10个比特币，亏损0.1个比特币，亏损1%；
  • 如果A用5倍的杠杆，则本金为2个比特币，亏损0.1个比特币，亏损5%；
  • 如果A用10倍的杠杆，则本金为1个比特币，亏损0.1个比特币，亏损10%；
  • 如果A用20倍的杠杆，则本金为0.5个比特币，亏损0.1个比特币，亏损20%；
  • 如果A用50倍的杠杆，则本金为0.2个比特币，亏损0.1个比特币，亏损50%；

（注：以上为方便说明，未考虑交易手续费因素的影响）

挂牌上市

1、交易所正式上线交易首日

BTC-Q虚拟币期货交易所的比特币期货合约于2014年10月24日中午12点正式开盘交易。同时上市的四份期货合约是：

BTC1410、BTC1411、BTC1412、BTC1503

根据11:55分的比特币价格指数确定新开合约的杠杆系数并进行公布。

每份合约新开盘形成第一笔成交价前，除要求报价须在价格指数5%涨跌幅以内外，不做其他限制。第一笔成交价形成后，对报价进行双重限制，具体参见《交易规则——报价》。

上市24小时内无成交的合约，24小时后的报价基准为最近合约实时最新成交价（如1411无成交，而1410的实时成交价为800，那么1410合约的报价基准价就为800）。最近合约无成交的，向前推到有成交的那个合约为止。

2、后续新合约的上市

后续新合约的上市时间：老合约退市的当天下午15:15分

后续新合约的杠杆系数：依据“孰高原则”进行确定，详见《交易规则-持仓盈亏计算》

后续新合约挂牌基准价：为距离该新合约最近的那个老合约在新合约上市时刻的实时最新成交价。

例：BTC1409合约的交割日是2014-9-30日，在BTC1409到期前的四份合约是：

BTC1409、BTC1410、BTC1412、BTC1503

BTC1409交割后，交易的四份合约是：

BTC1410、BTC1411、BTC1412、BTC1503

BTC1411为新上市的合约。

BTC1409在9月30日下午15:00准时交割。

BTC1411在交割15分钟后，即15:15分上市。上市的挂牌基准价为BTC1410在BTC1411上市时刻的实时最新成交价。

后续新合约开盘形成第一笔成交价前，除要求报价须在挂牌基准价5%涨跌幅以内外，不做其他限制。第一笔成交价形成后，对报价进行双重限制，具体参见《交易规则——报价》

报价

报价是指您可根据自己对市场走向的判断进行开多仓（看涨）、开空仓（看跌）的操作；相应地，您也可以自主进行平多仓、平空仓的操作。但为了遏制突发异常价格波动对市价产生的不良影响，我们对您的开仓和平仓报价作出了如下强制性规定：

1、报价限制——防止价格短时间内大幅波动

做多或平空时：最高可写入价为60秒前最新成交价+1%美元

做空或平多时：最低可写入价为60秒前最新成交价-1%美元

2、浮动涨跌停板制度——防止期货价格短时间内过度偏离现货价格

规定合约正常交易价格偏离价格指数一般不超过5%；如连续1个小时出现价格涨停顶格或者跌停顶格（即价格偏离价格指数的幅度一直保持在最大值），则此时启动“价格幅度的单向偏离增加5%，反向偏离幅度上调5%”，以此类推。

举例说明：以比特币价格上涨为例，假定当前比特币价格指数为500美元，当月合约价格涨到5%即525美元后，如果连续1个小时指数停留在500美元且价格停留在525美元（或者价格偏离指数的幅度始终在5%的比例），那么涨停价格增加5%，即价格可以偏离指数的幅度达到10%，此时反向的跌停价格增加5%（即如果涨停变为550，那么跌停为500，不是475也不是450，即涨停和跌停的价格差始终保持在10%以内）。如果价格下跌到500，且连续1个小时保持在这个幅度，那么跌停价格下调5%（即到475），此时涨停到了525，等于价格和指数的关系恢复正常。

止盈止损

止盈：该功能可以使用户在价格达到一个阀值的时候，选择以某个价格“获利了结平仓”，以实现“固定盈利”的一种智能化功能。用户可以根据自己的意愿设定止盈价格，系统固定0.1%的价格差作为触发价格。

“止盈功能”举例说明： 如果某用户以580美元的价格开多仓，那么该用户可以设定在价格达到600美元进行平仓操作，系统固定0.1%的差值后计算出的触发价格为600.6，即当价格达到600.6美元以后，系统自动挂出600美元的平仓单，此种情况下的平仓成为“止盈”。

止损：该功能可以使用户在价格达到一个阀值的时候，选择以某个价格“认亏平仓”，以实现“控制亏损”的一种智能化功能。

“止损功能”举例说明：如果某用户以600美元的价格开多仓，假定是以10倍杠杆的保证金交易，那么该用户的爆仓价格是540美元，则该用户可以在其爆仓价格540美元之上设定 “挂价580平仓”，系统固定0.1%的差值后计算出的触发价格为580.58，即当价格达到580.58美元以后，系统自动挂出580美元的平仓单，此种情况下的平仓成为“止损”。

需要说明的是：

  1. 为了防止您的挂单价和市价偏离过大导致系统性风险，我们限定您不必填写触发价格，我们自动取您的意向挂单价格0.1%的偏离价格作为触发价格；
  2. “止损”和“止盈”功能其中一个触发时，另一个功能失效。例如，您挂的单子“止盈”触发以后，您的单子会挂出去，此时无论该笔单子全部成交还是部分成交，系统均不再自动执行撤单操作，相应的“止损”功能失效（除非您手动撤单，再次重新设定止盈止损）；“止损”先触发时同理；
  3. 止盈和止损功能只对当前可用持仓有效；
  4. 为了降低系统风险，系统只负责挂价，不代表价格挂出后一定能成交，特别是在跳水行情发生时，所以用户应当对该功能有充分的认识，不能过度依赖该功能。

爆仓

BTC-Q期货合约采用“保证金爆仓制度”，即同一个品种、同一个合约、同一方向的持仓，系统将按照用户的持仓均价和最新成交价的差值比例计算实际盈亏，在最新成交价击穿用户的所有保证金金额的情况下，系统自动按照保证金亏损100%的价格对用户的穿仓头寸进行强制的挂出买单（做空的情况下）或者卖单（做多的情况下）。请注意，只挂单，不管是否成交，系统不自动砸价格，以免造成系统风险。爆仓不收手续费。一旦用户的该仓位被挂出去，该用户即丧失对该笔交易的处分权利。

自动追加保证金

此功能默认为“未开启状态”，用户可以通过点击“自动追加保证金”按纽进行开启/关闭。此功能可以使用户利用账户“可用余额+相应合约的待结算盈利金额（优先使用待结算盈利金额）”，在用户所持仓位的亏损达到或超过50%时，自动追加保证金金额50%的比例的金额到保证金账户（追加的保证金不收手续费）

请注意，自动追加保证金功能需满足以下几个触发条件：

  1. 您已开启自动追加保证金功能
  2. 账户内“可用余额+相应合约的待结算盈利金额（优先使用待结算盈利金额）”足以支付一次持仓保证金50%的保证金

    例如，某用户开仓了10个比特币，用了1个币的保证金（即用的10倍杠杆），那么该用户每次追加的保证金数量为保证金金额的50%，即0.5个比特币。也就是说，追加的保证金金额，只取决于您的实际使用保证金金额，不受您使用的杠杆倍数的影响。

需要强调说明的是：

  1. 追加的保证金从您的“可用余额+相应合约的待结算盈利金额”之和中扣除，并且优先使用待结算盈利金额；
  2. 追加的保证金不收取手续费；
  3. 追加的保证金在您平仓之后才会返回；
  4. 在您有多个合约持仓时，按照您持仓合约的开仓时间顺序先后追加。

交割

交割是指在合约到期而用户又未提前选择平仓了结的情况下，交易所将对到期的合约进行强制交割。

交割时间：每个合约月份最后一天的下午15:00整开始，到15:15分结束；

交割价格：以合约最后交易时刻15:00整的比特币价格指数为准。

系统亏损均摊

在不采用传统的期货交易所中向所有参与交易用户征收风险准备金办法的情况下，为了防范交易所交割的系统性风险，在每份合约完成交割前，您在该合约上的盈利计入该月份的“待结算盈利”账户中，以便于对合约期内未成交的爆仓单（爆仓后挂入市场的交易单，未成交相当于交易所持有）产生的亏损进行分摊。

我们保留了现有数字货币期货交易模式中，在交割完成后，对合约期内未成交的爆仓单产生的亏损在最后合约退市的交割完成后，按比例在所有在该份合约上产生净盈利的用户帐户中分摊扣除，即在该份合约的“待结算盈利”账户中分摊扣除的模式。

比如交割后某合约的所有帐户净盈利总计有100BTC，净亏损为99BTC，则差值的1BTC就是未成交爆仓单所产生的亏损，则系统将在该份合约待结算盈利账户为正值的用户中分摊扣除 （100-99）/100 = 1% 的分摊。例如，假设某用户在此份合约交割时的待结算盈利为10BTC,则会扣除10*1%=0.1BTC。

此外，为了提高用户的资金周转效率，除了到最后交割时刻的盈利分摊方式之外，我们给用户提供了实时分摊结算的方式，即用户可以在每份合约的最后交割时刻到来之前的1个小时外的任意时间内，主动申请提前结算，提前结算时，系统将按照用户申请时刻的系统亏损均摊比例数值对用户的待结算盈利申请进行分摊结算，实时结算完成后，用户待结算盈利账户分摊后剩余的金额将添加到用户的可用余额当中，用户可以用来提现或者在其他月份的合约上进行交易。

注1：某一合约的“待结算盈利账户”中的金额在该份合约交割分摊完成后或者在用户申请实时结算完成后，会添加到用户的可用余额当中，此时用户可以提现，也可以用于其他月份合约的开仓交易。在没有进行交割分摊或者申请实时结算之前，用户可以利用该账户中的金额进行该月份的开仓或追加保证金，但不可以提现，亦不可以使用到其他月份合约的开仓交易之中。

注2：某一合约的“系统亏损均摊比例”=该合约系统亏损剩余未分摊金额÷（该合约待结算盈利账户总和+该合约用户持仓盈利总和）

其中：该合约系统亏损剩余未分摊金额=用户申请结算时刻系统被动持仓（即用户被系统挂出去的未成交的爆仓单）按照平台实时最新成交价计算的亏损额，减去在该份合约上提前申请实时结算的用户已经分摊的金额。
